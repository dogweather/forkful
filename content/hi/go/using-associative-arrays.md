## क्या और क्यों?

गो में, संयोजक सरणियों को मैप्स के रूप में जाना जाता है, जो आपको कुंजी-मान जोड़ों के साथ डेटा संग्रहित करने और प्राप्त करने की अनुमति देते हैं। जब आपको एक अद्वितीय कुंजी द्वारा त्वरित रूप से मान देखने की आवश्यकता होती है, तब वे संग्रह का प्रबंधन आसान बनाते हैं, आपके प्रोग्राम में डेटा प्रबंधन और पुनःप्राप्ति को सरल बनाते हैं।

## कैसे:

गो में, मैप्स का उपयोग करना बहुत सरल है। यहाँ चीजों को शुरू करने के लिए एक सरल मार्गदर्शिका है:

1. **मैप्स की घोषणा और प्रारंभिकरण**

```Go
package main

import "fmt"

func main() {
    // एक खाली मैप को प्रारंभ करता है जिसमें स्ट्रिंग कुंजियाँ और इंट मान हैं
    var scores map[string]int
    fmt.Println(scores) // प्रिंट्स: map[]

    // एक गैर-खाली मैप की घोषणा और प्रारंभिकरण
    colors := map[string]string{
        "red": "#ff0000",
        "green": "#00ff00",
    }
    fmt.Println(colors) // प्रिंट्स: map[green:#00ff00 red:#ff0000]
}
```

2. **तत्वों को जोड़ना और पहुँचाना**

```Go
func main() {
    fruits := make(map[string]int)
    fruits["apples"] = 5
    fruits["bananas"] = 10

    fmt.Println(fruits["apples"]) // प्रिंट्स: 5
}
```

3. **मैप्स के ऊपर इटरेट करना**

```Go
func main() {
    pets := map[string]string{"dog": "bark", "cat": "meow"}

    for key, value := range pets {
        fmt.Printf("%s goes %s\n", key, value)
    }
    // आउटपुट क्रम भिन्न हो सकता है, क्योंकि मैप्स क्रम की गारंटी नहीं देते।
}
```

4. **तत्वों को हटाना**

```Go
func main() {
    meals := map[string]int{"breakfast": 300, "lunch": 600}
    fmt.Println(meals) // हटाने से पहले

    delete(meals, "lunch")
    fmt.Println(meals) // हटाने के बाद
}
```

## गहराई में जानकारी

गो 1 में परिचय कराया गया, मैप्स संयोजक सरणियों को कुशलता से संभालने का एक निर्मित तरीका प्रदान करते हैं। स्लाइस के विपरीत, जो क्रमबद्ध संग्रह हैं, मैप्स क्रमबद्ध नहीं होते। इसका मतलब है कि मैप तत्वों पर इटरेशन का क्रम प्रत्येक निष्पादन में समान नहीं हो सकता, कुंजी-मान जोड़ों को लचीलेपन के साथ गतिशील रूप से संभालने की इसकी क्षमता के लिए एक समझौता।

मूल रूप से, गो मैप्स को हैश टेबल के रूप में लागू करता है, यह सुनिश्चित करता है कि पहुँच, सम्मिलन, और हटाने के ऑपरेशन की औसत जटिलता अधिकतर परिस्थितियों में O(1) है। हालाँकि, हैश टकरावों जैसे कारकों के आधार पर यह कुशलता भिन्न हो सकती है।

क्रमबद्ध कुंजी यात्रा की आवश्यकता वाले उपयोग के मामलों के लिए, आप मैप्स को स्लाइस के साथ संयोजन करने या क्रमबद्ध मैप्स या वृक्ष जैसी अतिरिक्त डेटा संरचनाएँ प्रदान करने वाले तृतीय-पक्ष पैकेजों का परीक्षण करने पर विचार कर सकते हैं। इनकी सीमाओं के बावजूद, गो के मैप्स कई प्रोग्रामिंग परिदृश्यों के लिए एक शक्तिशाली और आवश्यक उपकरण हैं।
