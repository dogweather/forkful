## क्या और क्यों?
डीबगर का उपयोग करना जैसे कोड के जंगल में GPS होना; यह आपको समस्या के स्रोत तक निर्देशित करता है। प्रोग्रामर डीबगर्स का उपयोग अपने कोड के माध्यम से कदम बढ़ाने, वेरिएबल्स का निरीक्षण करने और प्रवाह को समझने के लिए करते हैं, जो बग्स को पकड़ना और प्रदर्शन को ऑप्टिमाइज़ करना आसान बनाता है।

## कैसे:
Go के पास Delve (`dlv`) नामक एक बिल्ट-इन टूल है जो डीबगिंग के लिए है। शुरू करने के लिए, Delve को इंस्टॉल करें, एक साधारण Go प्रोग्राम लिखें, और फिर इसे डीबगर के माध्यम से चलाएं।

```Go
// पहले, Delve को इंस्टॉल करें
// go get -u github.com/go-delve/delve/cmd/dlv

// उदाहरण Go प्रोग्राम, main.go के रूप में सेव करें
package main

import "fmt"

func main() {
    message := "Debugging with Delve!"
    fmt.Println(message)
}

// अपने प्रोग्राम को Delve के साथ चलाएं
// dlv debug

// कुछ बुनियादी Delve कमांड:
// (dlv) break main.main // मेन फंक्शन में एक ब्रेकपॉइंट सेट करें
// (dlv) continue // ब्रेकपॉइंट या प्रोग्राम समाप्ति तक चलाएं
// (dlv) step // प्रोग्राम के माध्यम से एकल कदम उठाएँ
// (dlv) print message // वेरिएबल 'message' का वर्तमान मूल्य प्रिंट करें
// (dlv) quit // Delve से बाहर निकलें
```

`dlv debug` चलाने से एक डीबगिंग सत्र शुरू होता है। एक बार जब आप एक ब्रेकपॉइंट पर पहुंच जाते हैं जिसे आपने सेट किया है, तो आप अपने प्रोग्राम के माध्यम से कदम उठा सकते हैं और देख सकते हैं कि हुड के नीचे क्या हो रहा है।

## गहन पड़ताल
ऐतिहासिक रूप से, Go प्रोग्रामरों ने जैसे कि GDB (GNU Debugger) लेकिन GDB का सामना चुनौतियों से हुआ क्योंकि GDB, Go के रनटाइम और गोरुटिन्स के लिए बना नहीं था। Delve बेहतर समर्थन के साथ Go की अनूठी विशेषताओं के लिए मदद को आया।

Delve के विकल्प के रूप में `go-dbg`, और यहां तक कि Visual Studio Code और GoLand जैसे IDE में इंटीग्रेटेड डीबगर समर्थन भी है, जो अधिक उपयोगकर्ता-अनुकूल अनुभव के लिए Delve के आसपास लपेटते हैं।

कार्यान्वयन के पक्ष में, Delve `runtime` और `debug/gosym` पैकेजों के साथ दूसरों के बीच, Go प्रोग्राम सिंबोल्स और रनटाइम जानकारी को एक्सेस करने और व्याख्या करने के लिए काम करता है। यह नई भाषा विशेषताओं और संस्करणों के साथ अपडेट रखे जाने के लिए लगातार अपडेट किया जाता है।
