## Що це та навіщо?
Перевірка існування директорії - це спосіб дізнатися, чи існує певний шлях у файловій системі. Програмісти роблять це, щоб уникнути помилок при спробах доступу чи модифікації файлів у неіснуючих каталогах.

## Як це робити:
В Go перевірка на існування директорії вимагає використання пакету `os`. Ось простий приклад коду:

```Go
package main

import (
    "fmt"
    "os"
)

func main() {
    path := "./назва_вашої_директорії"

    if _, err := os.Stat(path); os.IsNotExist(err) {
        fmt.Printf("Директорії %s не існує.\n", path)
    } else {
        fmt.Printf("Директорія %s існує.\n", path)
    }
}
```

При запуску коду отримаємо такий результат:

```
Директорія ./назва_вашої_директорії існує.
```
або
```
Директорії ./назва_вашої_директорії не існує.
```

## Поглиблений огляд
Перевірка на існування директорії була завжди важливою частиною програмування, оскільки неправильне управління файлами може спричинити серйозні помилки. У Go, `os.Stat` повертає два значення: `os.FileInfo`, яке описує файл, і `error`, який буде не `nil`, якщо виникла помилка. Функція `os.IsNotExist(err)` визначає чи помилка вказує на відсутність файлу чи директорії.

Альтернативою може бути використання `os.Lstat`, яка не переходить символічні посилання. Це може бути корисно для специфічних завдань, де символічні посилання не слід розглядати як цільові директорії.

Щодо реалізації, `os.Stat` інкапсулює системні виклики, які відрізняються залежно від операційної системи, але Go абстрагує ці відмінності, надаючи уніфікований API для розробників.
