## Що та чому?
Округлення чисел — це відкидання цифр після певного місця з можливістю опціонального корегування останньої збереженої цифри. Програмісти округляють числа для зменшення точності, коли точні значення не є необхідними, управління помилками з плаваючою точкою або підготовки чисел для зручного відображення користувачам.

## Як:
В C, зазвичай використовують функції `floor()`, `ceil()`, чи `round()`. Ось короткий приклад:

```C
#include <stdio.h>
#include <math.h>

int main() {
    double num = 3.14159;
    double num_floor = floor(num);
    double num_ceil = ceil(num);
    double num_round = round(num);

    printf("Floor: %.2f\n", num_floor); // Floor: 3.00
    printf("Ceil: %.2f\n", num_ceil);   // Ceil: 4.00
    printf("Round: %.2f\n", num_round); // Round: 3.00
    return 0;
}
```

Для більшого контролю, наприклад округлення до певного місця, ви множите, округляєте та ділите:

```C
double roundToPlace(double num, int place) {
    double scale = pow(10.0, place);
    return round(num * scale) / scale;
}

// ...

double num = 3.14159;
double num_rounded = roundToPlace(num, 2);
printf("Rounded to 2 decimal places: %.2f\n", num_rounded); // Округлено до 2 десяткових місць: 3.14
```

## Поглиблений огляд
У минулі часи округлення часто означало ручну роботу — велике навантаження з простим олівцем та папером. З появою обчислень ми автоматизували цей процес, але арифметика з плаваючою точкою принесла нюанси через її бінарну природу, де деякі числа не можуть бути точно представлені.

Альтернативи стандартному округленню включають усічення (просте відкидання зайвих цифр) або банківське округлення, яке округлює до найближчого парного числа, коли значення знаходиться точно між двома значеннями, знижуючи упередженість у повторюваних розрахунках.

Реалізація стає складною, коли потрібно округлити числа з довільною точністю або обробити спеціальні випадки, як-от нескінченність, сигнальні NaNs чи субнормальні значення. Функції стандартної бібліотеки C обробляють основи, але якщо вам потрібно округлити десяткові числа особливим чином, потрібно буде більше, ніж `math.h`.
