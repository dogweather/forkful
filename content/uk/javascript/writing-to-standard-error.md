## Що це та Навіщо?

Стандартний потік помилок (stderr) — місце для логування помилок та діагностики. Програмісти використовують stderr, щоби відділяти звичайний вивід програми від повідомлень про помилки, таким чином полегшуючи налагодження та моніторинг.

## Як це робити:

В JavaScript для запису у stderr можна використовувати глобальний об’єкт `process` у Node.js. Ось як це виглядає на практиці.

```javascript
// Запис у стандартний потік помилок
process.stderr.write('Це повідомлення помилки.\n');

// Або використовуючи console.error
console.error('Це теж повідомлення помилки.');
```

Очікуваний результат у консолі:

```
Це повідомлення помилки.
Це теж повідомлення помилки.
```

## Поглиблено:

`stderr` використовується з часів першісних операційних систем Unix. Відокремлення `stdout` (стандартний вивід) від `stderr` дозволяє перенаправляти помилки окремо від основного виводу, наприклад, у файл або інший обробник. В Node.js, `process.stderr` є потоком запису, таким як `process.stdout` для звичайного виводу. Обидва вони – частина інтерфейсу потоків, і надають методи для роботи з даними асинхронно.

Альтернативним способом логування помилок може бути використання зовнішніх бібліотек або власних обгорток, які забезпечують додаткові функціональності, такі як форматування чи збереження журналів у файл.
