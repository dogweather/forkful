## Що це таке та навіщо?
Запис в стандартний потік помилок (stderr) дозволяє відокремити звичайний вивід програми від повідомлень про помилки. Програмісти використовують stderr для діагностики та логування помилок, що дозволяє легко фільтрувати та аналізувати проблеми.
## Як це зробити:
```Swift
import Foundation

// Звичайний вивід
print("Це звичайний вивід.")

// Вивід помилки
fputs("Це повідомлення помилки.\n", stderr)

```
**Приклад виводу:**
```
Це звичайний вивід.
Це повідомлення помилки.
```
## Поглиблений розгляд:
Стандартні потоки були стандартизовані UNIX системами для спрощення обробки виводу програм. Альтернативою `fputs` є створення `FileHandle` для stderr, але `fputs` є більш прямолінійним. На низькому рівні, stderr є файловим дескриптором, який програми використовують для запису помилок, так що вони не змішуються зі стандартним виводом (stdout).
