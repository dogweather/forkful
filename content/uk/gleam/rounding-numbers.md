## Що та Чому?
Округлення чисел полягає у коригуванні значення до найближчого вказаного місця — наприклад, з 2.56 до 3, якщо ми округлюємо до цілих чисел. Програмісти роблять це для спрощення або щоб задовольнити певні числові специфікації, зазвичай, щоб уникнути нюансів, спричинених помилками точності з плаваючою комою, або зробити вивід зручнішим для користувача.

## Як це робити:
У Gleam, на момент мого останнього перевірки, округлення не входить до стандартної бібліотеки, але ось як ви зазвичай округлите число з плаваючою комою до найближчого цілого числа, використовуючи прямо функції Erlang:

```gleam
external fn erlang_round(Float) -> Int = "erlang" "round"

pub fn main() {
  let rounded = erlang_round(2.56)
  rounded // Вивід: 3
}
```

Вивід:
```
3
```

Маєте на увазі іншу точність? Скажімо, округлення до двох десяткових місць? Нам знадобиться трохи математики:

```gleam
pub fn round_to_two_places(num: Float) -> Float {
  let multiplier = 100.0
  let tmp = num * multiplier
  let round_tmp = erlang_round(tmp)
  round_tmp / multiplier
}

pub fn main() {
    round_to_two_places(2.569) // Вивід: 2.57
}
```

Вивід:
```
2.57
```

## Поглиблений огляд
Історично, округлення чисел завжди було важливим, особливо у фінансових та наукових обчисленнях, де точність та стандарти мають велике значення. Без округлення ми б отримали неприємно довгі десяткові числа скрізь, що робило б обчислення непрактичними та схильними до помилок.

У світі програмування різні мови пропонують різні підходи, від вбудованих функцій до всебічних математичних бібліотек. Округлення може включати різні правила — наприклад, "округлення вгору" (звичайний метод) або "округлення до парного" (часто використовується у фінансових розрахунках, щоб уникнути упередженості).

Gleam, будучи молодою мовою з корінням у Erlang, покладається на надійний набір числових функцій Erlang. Як мова розвиватиметься, ми можемо побачити введення нативних функцій, що зменшить потребу викликати зовнішні рутини.
