## Що і чому?
REPL (цикл "читати-оцінити-вивести") - це просте, інтерактивне програмне середовище. Програмісти використовують його для експериментів з мовою в реальному часі, швидких завдань або для розуміння нових концепцій без необхідності створення повноцінних додатків.

## Як користуватися:
C++ не має вбудованого REPL, але інструменти на кшталт Cling пропонують таку можливість. Ось як використовувати Cling для розрахунку суми двох чисел:

```C++
#include <iostream>

int main() {
    int a = 5;
    int b = 7;
    std::cout << "Сума становить: " << a + b << std::endl;
    return 0;
}

// Вивід:
// Сума становить: 12
```

Запустіть Cling і вводьте код рядок за рядком, спостерігаючи за виводом після кожної команди. Це негайна відповідь, без компіляції.

## Поглиблений огляд
REPL поширені для мов, таких як Python або Lisp, і вони існують з 1960-х років. Для C++, компільованої мови, концепція не підходить так природно, саме тому існують інструменти на кшталт Cling — вони інтерпретують C++ на льоту. Альтернативи включають онлайн компілятори або маломасштабні тестові програми, скомпільовані традиційним способом. Cling побудований на основі LLVM та Clang, забезпечуючи місток для використання C++ в інтерпретованому стилі.

## Дивіться також
- [Cling](https://root.cern/cling/): Інтерактивний інтерпретатор C++, створений на основі бібліотек LLVM та Clang.
- [Jupyter Notebooks](https://jupyter.org/): Пропонує інтерактивну оболонку у середовищі ноутбука, підтримує C++ через ядро xeus-cling.
- [LLVM](https://llvm.org/): Колекція модульних та використовуваних повторно технологій компілятора та інструментального ланцюга, на яких базується Cling.
