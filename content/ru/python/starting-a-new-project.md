## Что и Почему?

Начало нового проекта - это создание нового каталога с файлами, подготовленными для вашего нового кодового приключения. Это как начало строительства на стройплощадке, только для программистов. Мы делаем это, чтобы превратить идеи в работающее программное обеспечение, организовать наш код и управлять сложностью с самого начала.

## Как это сделать:

Давайте начнем проект на Python. Сначала создадим новый каталог:

```bash
mkdir my_new_project
cd my_new_project
```

Теперь настроим виртуальное окружение - это позволит держать зависимости нашего проекта в порядке:

```bash
python -m venv venv
source venv/bin/activate # На Windows используйте `venv\Scripts\activate`
```

Подготовив нашу виртуальную землю, посадим семена вашего проекта с файлом `main.py`:

```bash
touch main.py
echo "print('Hello, new project!')" > main.py
python main.py
```

Вывод:
```plaintext
Hello, new project!
```

Для надежности давайте закрепим зависимости на раннем этапе. Даже если их пока нет:

```bash
pip freeze > requirements.txt
```

И это эмбрион вашего проекта. Отсюда он будет расти.

## Глубокое погружение

В прошлом многие программисты просто начинали кодить в одиночном файле. Часто это приводило к хаосу по мере роста проекта. В наши дни у нас есть лучшие практики.

Для Python у нас есть соглашения, такие как PEP 8 для руководства по стилю. Есть также инструменты, например `cookiecutter`, которые создают проекты из шаблонов. Хотите веб-приложение? Для этого есть шаблон. Это создано, чтобы сэкономить ваше время.

С другой стороны, вам может понравиться делать это вручную, как мы показали выше. Этот метод дает вам полный контроль, позволяя построить ваш проект с нуля. Просто не забудьте отслеживать зависимости с помощью `requirements.txt`. Это критически важно, когда вы делитесь вашим проектом или развертываете его.
