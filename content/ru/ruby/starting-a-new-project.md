## Что и Почему?

Начало нового проекта похоже на посадку семени в ваш цифровой сад – вы запускаете свежий набор идей и превращаете их в код, который выполняет что-то полезное. Программисты начинают новые проекты для решения проблем, исследования концепций или просто ради чистого удовольствия от создания чего-то нового.

## Как это сделать:

Итак, вы готовы превратить эти идеи в проект на Ruby? Поехали. Начнем с основ.

```Ruby
# Установите Ruby, если вы еще этого не сделали.
# Проверьте вашу версию Ruby, чтобы убедиться, что она актуальна:
$ ruby -v

# Вывод должен показать текущую версию Ruby:
# ruby 3.x.x

# Теперь давайте создадим директорию для вашего проекта:
$ mkdir my_new_project
$ cd my_new_project

# Теперь инициируйте новый репозиторий Git, если вы хотите контроль версий (настоятельно рекомендуется):
$ git init

# Затем создайте файл для входа, назовем его 'app.rb':
$ touch app.rb

# Начинайте программировать! Напишите простой вывод, чтобы убедиться, что все работает:
puts "Hello New Project!"

# Запустите ваш файл:
$ ruby app.rb

# Вывод должен быть:
# Hello New Project!
```

## Подробнее

В прошлом начать новый проект на Ruby было немного грубо – только вы, текстовый редактор и куча файлов `.rb`. По мере развития языка появились инструменты, упрощающие этот процесс.

Например, Bundler управляет вашими гемами – библиотеками Ruby – так что вы можете отслеживать и устанавливать зависимости с легкостью. Просто запустите `bundle init` после настройки директории вашего проекта, чтобы создать `Gemfile`, где вы будете перечислять ваши гемы.

Затем у нас есть менеджер версий Ruby (RVM) и окружение Ruby (rbenv), которые помогают переключаться между версиями Ruby для проекта. Очень удобно, если вы работаете со старым кодом.

А что насчет фреймворков? Ruby on Rails – это большое имя для веб-приложений. Но если вам нужно что-то легковесное (например, для сервисов или API), обратите внимание на Sinatra или Roda.
