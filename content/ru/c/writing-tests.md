## Что и Почему?

Написание тестов означает создание кода, который проверяет, работает ли ваш другой код. Программисты делают это, чтобы заранее выявлять ошибки, экономить время и убедиться, что код работает сейчас и будет работать и в будущем.

## Как:

В языке C вы можете писать тесты, используя фреймворк для тестирования, такой как CUnit, или создав свою простую систему тестирования. Ниже приведен простейший пример использования assert для создания базовой тестовой функции для функции `add`.

```C
#include <assert.h>

// Функция для тестирования
int add(int a, int b) {
    return a + b;
}

// Тестовая функция
void test_add() {
    assert(add(2, 2) == 4);
    assert(add(-1, 1) == 0);
}

int main() {
    test_add();
    printf("Все тесты пройдены!\n");
    return 0;
}
```

Пример вывода, если все тесты пройдены:

```
Все тесты пройдены!
```

Если тест не пройден, программа будет прекращена и выведено сообщение об ошибке.

## Глубокое погружение

Исторически C не имел встроенного фреймворка для тестирования. Программисты обычно писали собственные тестовые функции или использовали сторонние фреймворки. Среди популярных фреймворков: CUnit, Check и Unity. Каждый предлагает такие функции, как автоматическое обнаружение тестов, процессы настройки/разрушения и отчеты о результатах тестов. Для малых проектов могут подойти простые тесты на основе assert, но по мере роста сложности подходящий фреймворк экономит время и усилия.
