## อะไรและทำไม?

การสร้างไฟล์ชั่วคราวหมายถึงการทำไฟล์สำหรับการใช้งานระยะสั้น โปรแกรมเมอร์ทำเช่นนี้เพื่อเก็บข้อมูลที่จำเป็นเฉพาะระหว่างการทำงานของโปรแกรม เช่น ผลลัพธ์ระหว่างกาล หรือเพื่อให้แน่ใจว่ามีสถานะที่สะอาดโดยไม่ทำให้พื้นที่เก็บข้อมูลถาวรเกะกะ

## วิธีทำ:

ใน Fish Shell, คุณสามารถสร้างไฟล์ชั่วคราวโดยใช้ `mktemp` นี่คือตัวอย่างอย่างรวดเร็ว:

```fish
set tempfile (mktemp)
echo "Hello, temporary world!" > $tempfile
cat $tempfile
rm $tempfile
```

และคุณจะเห็นบางอย่างเช่นนี้:

```shell
Hello, temporary world!
```

นี่สร้างไฟล์ชั่วคราว นำข้อความเขียนลงไป แสดงเนื้อหา และจากนั้นลบไฟล์

## ลงลึก

ในอดีต ไฟล์ชั่วคราวมักจะถูกสร้างขึ้นด้วยตนเอง นำไปสู่ปัญหาการชนกันของชื่อไฟล์และปัญหาด้านความปลอดภัย `mktemp` ช่วยให้รอด! คำสั่งนี้สร้างไฟล์ที่มีชื่อเฉพาะตัว ลดความเสี่ยงของการชนของไฟล์

วิธีการอื่นๆ รวมถึงการเขียนลง `/dev/shm` บน Linux หรือการใช้งานไฟล์สิทธิ์ตามหน่วยความจำ อย่างไรก็ตาม วิธีเหล่านี้ไม่มีความพกพาเท่ากับ `mktemp`

สำหรับอายุขัยของไฟล์ชั่วคราว เป็นเรื่องสำคัญที่ต้องจำไว้ว่าควรจะถูกลบโดยโปรแกรมที่สร้าง นี่เป็นการรับประกันว่าไม่มีไฟล์เหลือที่ใช้พื้นที่ของระบบ ในบางระบบ ไดเร็กทอรี `/tmp` จะถูกลบที่การรีบูต แต่คุณไม่ควรพึ่งพาพฤติกรรมนี้เพื่อการล้างข้อมูล
